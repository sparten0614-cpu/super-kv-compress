\section{Experimental Methodology}
\label{sec:method}

This section describes our experimental design, covering the models, compression configurations, and evaluation protocol used throughout the study.

\subsection{Compression Configurations}

\subsubsection{Quantization}

We evaluate KV cache quantization at multiple bit-widths using llama.cpp's GGUF quantization types:

\begin{itemize}
    \item \textbf{q8\_0}: 8-bit block quantization (2$\times$ compression)
    \item \textbf{q5\_1}: 5-bit with offset (3.2$\times$ compression)
    \item \textbf{q5\_0}: 5-bit block quantization (3.2$\times$ compression)
    \item \textbf{q4\_1}: 4-bit with offset (4$\times$ compression)
    \item \textbf{q4\_0}: 4-bit block quantization (4$\times$ compression)
    \item \textbf{TQKV\_6}: TurboQuant 6-bit with Hadamard rotation (2.67$\times$ compression)
\end{itemize}

Crucially, llama.cpp supports \textbf{independent K and V type specification} via \texttt{--cache-type-k} and \texttt{--cache-type-v}, enabling asymmetric configurations (e.g., K=q8\_0, V=q4\_0). This is the only mainstream inference framework with this capability. When using flash attention on CUDA, the full $7 \times 7$ type combination matrix requires building with \texttt{-DGGML\_CUDA\_FA\_ALL\_QUANTS=ON}.

\subsubsection{Eviction}

We evaluate two eviction strategies:

\begin{itemize}
    \item \textbf{StreamingLLM}: Position-based eviction retaining the first 128 tokens (attention sinks) plus the most recent tokens, discarding all middle tokens.
    \item \textbf{H\textsubscript{2}O}: Attention-score-based eviction retaining tokens with the highest cumulative attention weights, regardless of position.
\end{itemize}

Eviction rates range from 0\% (no eviction) to 90\% in 10-percentage-point increments. The 1\% PPL cliff is identified as the eviction rate where PPL degradation first exceeds 1\%.

\subsubsection{Asymmetric K/V Diagnostic}

For models exhibiting quantization failure (Qwen), we run a full asymmetric diagnostic matrix:
\begin{itemize}
    \item K=FP16 with V=q4\_0
    \item K=q4\_0 with V=FP16
    \item K=q8\_0 with V=q4\_0
    \item K=q8\_0 with V=FP16
\end{itemize}
This isolates whether the failure originates from K-quantization, V-quantization, or their interaction.

\subsection{Evaluation Protocol}

\subsubsection{Perplexity (PPL)}

WikiText-2 perplexity, measured per-token using llama.cpp's \texttt{perplexity} tool. Default context length is 16K tokens with 2 chunks unless otherwise noted. PPL is reported as the absolute value and the relative change ($\Delta$PPL \%) versus the FP16 baseline.

\subsubsection{Needle-in-a-Haystack (NIAH)}

A synthetic retrieval benchmark testing whether the model can locate and reproduce a specific fact embedded at a controlled position within a long context. Our implementation:

\begin{itemize}
    \item \textbf{Needle}: ``The secret code for Project Alpha is 7492-BLUE-DELTA.''
    \item \textbf{Haystack}: Unrelated filler text to specified context length
    \item \textbf{Positions}: 10\%, 25\%, 50\%, 75\%, 90\% of context
    \item \textbf{Scoring}: Exact substring match in the \emph{generated text only} (excluding prompt echo)
    \item \textbf{Accuracy}: Fraction of positions where the needle is successfully retrieved
\end{itemize}

NIAH directly tests the \emph{causal limitation} of eviction: tokens at arbitrary positions must survive compression for retrieval to succeed.

\subsubsection{LongBench v2}

A multi-task long-context benchmark with 20 questions spanning QA, summarization, and few-shot learning. Provides a task-based evaluation complementary to PPL.

\subsection{Outlier Detection}

For each model, we profile the maximum absolute value of K projections across layers using a calibration run. Layers with $K_\text{max} > 50$ are flagged as outlier layers. These outliers dominate the quantization range in low-bit formats, destroying precision for normal-magnitude entries. The mitigation is to skip outlier layers from quantization (retain FP16 KV for those layers).

\subsection{Hardware and Reproducibility}

All experiments use llama.cpp (latest main branch) with GGUF model files. Quantization perplexity experiments run on a cloud RTX 5880 (48GB VRAM); NIAH and eviction experiments on both RTX 5880 and Apple Mac Mini M4 (16GB). Scripts for all benchmarks are provided in the companion repository.\footnote{\url{https://github.com/sparten0614-cpu/super-kv-compress}}
